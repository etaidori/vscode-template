# Workspace Agent Configuration

## Git Workflow Control

**Default Behavior:** Skip git operations (commit, push, version bumping) unless explicitly instructed.

**When to Run Git Workflow:**
- Only execute git workflow steps when the user includes "Process Git" in their request
- Git workflow includes: version bumping, npm package, git add, git commit, git push

**Allowed Operations Without "Process Git":**
- Code changes and edits
- Documentation updates
- Building/compiling
- Creating files and folders
- Running tests or validation

**Example Requests:**
- ❌ "Add a new feature" → Code changes only, no git
- ✅ "Add a new feature. Process Git" → Code changes + git workflow

## Version Management

When "Process Git" is included:
1. Update version in `package.json` 
2. Update documentation with changes
3. Run `npm run package` (handles versioning and VSIX output)
4. Stage changes: `git add -A`
5. Commit with intelligent prefix (feat:/fix:/chore:/docs:/refactor:)
6. Push to remote

## File Change Priority

Always update these files when making relevant changes (even without "Process Git"):
- `package.json` - for version numbers
- `template-readme.md` - for documentation changes
- Source files in `src/` and `templates/`
